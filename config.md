# Coordination constants

Single source of truth for the numeric constants used by the coordination machinery defined in `CLAUDE.md`, the cursor mirror, and the skills. Imported into every Claude Code session via `@~/.claude-agent/config.md` at the end of `CLAUDE.md`.

Edit values **here** — not in the prose that references them. Every cross-reference uses the **key**, not the literal value. Where a mechanical surface (spawn template, shell script) must embed a literal number, an inline comment names the key so `rg <key>` still finds the use site.

| Key | Value | Meaning |
|---|---|---|
| `max-recursion-depth` | `5` | Hard cap on `AGENT_RECURSION_DEPTH`. A spawn that would push the env var to a value **above** this is forbidden — escalate to the user instead. Applies to every `claude -p` invocation, including `overcome-difficulty`'s recursive escape. |
| `loop-sensitivity-depth` | `2` | At this depth and deeper, a spawned level must self-check whether its task is a re-framing of an ancestor's task and return `LOOP_DETECTED:` early rather than recursing further. |
| `budget-small-usd` | `1.00` | Expected-size **telemetry label** for small specialist work (single-file edit, narrow analysis, short plan refinement). NOT a kill-cap — under flat billing the dollar figure is list-price telemetry, not marginal money (leaf `flat-max-billing-cost-framing`). Recorded on the spawn-costs row and used for the soft-warn; the actual `--max-budget-usd` kill is `spawn-runaway-ceiling-usd`. |
| `budget-medium-usd` | `3.00` | Expected-size **telemetry label** for typical specialist work (multi-file change with tests, scoped refactor, standard plan). Default tier — pick when in doubt. Also the default for `overcome-difficulty`'s recursive escape. Label only, not a kill-cap (see `budget-small-usd`). |
| `budget-large-usd` | `8.00` | Expected-size **telemetry label** for cross-cutting changes (multi-stage plan, full feature, expensive research). Label only, not a kill-cap (see `budget-small-usd`); also the fail-safe fallback for `spawn-runaway-ceiling-usd` if that key is ever absent. |
| `spawn-runaway-ceiling-usd` | `25.0` | Single global **runaway backstop** passed as `--max-budget-usd` to every spawn, regardless of tier. Under flat billing the tier labels ($1/$3/$8) are NOT money limits; this ceiling exists only to kill a runaway/looping spawn — protecting the constraints that ARE scarce on flat (rate-limit headroom, wall-clock, attention). Set well above the large tier (~3×) so legitimate large work is never truncated. If this key is absent, spawn machinery falls back to `budget-large-usd` (never unbounded). |
| `small-change-max-lines` | `20` | Upper bound (line count) for the *small change* class in task triage. ≤ this many changed lines + single file + no architectural decision = manager may handle in-thread without a plan-approval gate. |
| `substantive-wall-clock-min` | `30` | Lower bound (minutes) for the *substantive* class in task triage — work expected to take this long or more warrants the full coordination cycle. |
| `rediscovery-threshold-min` | `5` | Quality bar for post-resolution experience leaves — record if skipping the leaf would cost a future similar task at least this much rediscovery. |
| `claude-md-max-lines` | `400` | Hard ceiling on `CLAUDE.md` line count. Above this, extract a section to a `memory-global/leaves/` leaf and reference it. Enforced by `scripts/lint-prose-length.py`. |
| `claude-md-max-chars` | `39500` | Hard ceiling on `CLAUDE.md` size in UTF-16 code units (chars) — the unit the harness itself measures (`content.length`), NOT UTF-8 bytes. The harness's own threshold is `max(40000, window * 0.05 * charsPerToken)` (function `dYr()` in the installed `@anthropic-ai/claude-code/bin/claude.exe` bundle), so the 40 000 floor binds only while the context window is pinned at or below ~266k via `CLAUDE_CODE_DISABLE_1M_CONTEXT`. Crossing it produces a DISPLAY-ONLY `/doctor` warning (the `large-memory-files` status card, built from `dYr()` → `pYr()` → `SYp()`) — there is NO truncation on the memory-file load path. Verified 2026-07-23 by reading the bundle and by an over-limit sentinel test (a tail sentinel past 40 000 chars was returned verbatim). This is a CLIENT-SIDE constant Anthropic can change in any release — re-check `dYr()` rather than re-derive this number. Enforced by `scripts/lint-prose-length.py` (the line-count guard does not catch char growth). |
| `readme-max-lines` | `140` | Hard ceiling on README.md line count — README is the thin entry-point; everything deeper lives under docs/. Enforced by scripts/lint-prose-length.py. |
| `cursor-mirror-max-lines` | `220` | Hard ceiling on `cursor/rules/claude-code-sync.mdc`. The mirror is thin by design. |
| `skill-md-max-lines` | `200` | Hard ceiling on any `skills/*/SKILL.md` or `skills/specializations/*/SKILL.md`. Detail belongs in a sibling `policy.md` or a memory leaf. |
| `policy-md-max-lines` | `400` | Hard ceiling on any `skills/*/policy.md`. Policies are allowed to be detailed but not unbounded. |
| `core-difficulty-mass-threshold` | `8` | Cluster mass (Σ severity over the severity-weight ladder low=1/medium=2/high=4/critical=8) at/above which a Core-difficulty cluster is flagged for a batched Core change (ADR-0001 § Difficulty-accumulation). Consumed by the digest `core-difficulty-digest.py`. **Basis:** `8` = four independent medium reports (4×2) or two high (2×4) — the recurrence point at which a difficulty has been reported enough to warrant a Core change rather than a one-off; any single critical short-circuits to flag regardless. Derived from the ladder, not guessed; recalibration guidance in `docs/architecture/core-difficulty-calibration.md` (ADR open question #2). |
| `difficulty-channel` | `github` | Channel for filing Core difficulties from non-author machines, **auto-detected** per machine by `scripts/difficulty_channel/detect.py` (strong internal Yandex signals → `startrek`; otherwise → `github`) and written to `~/.claude-agent/agent-identity.local` (`difficulty_channel=<name>`); override by editing that line. Values: `github` (GitHub Issues sthe0/claude-agent-instructions — the org-neutral default) or `startrek` (internal Yandex Tracker — opt-in, requires `~/.tracker-token`). The startrek channel files **reports** to `OOSEVENREPORT`; the org-internal **backlog** (proactive Yandex-specific refinements planned by org-authors) is the separate queue `OOSEVEN` — the 3-tier backlog/report model (and the per-project `instruction_queue` field) is in `memory-global/leaves/instruction-dev-queues.md`. Core commit authority is determined solely by `git push --dry-run` capability — no per-machine config flag (a shared config file cannot distinguish machines). |
| `advisor-mode` | `substantive` | Activation mode for the agentctl advisory judge (`advisor.resolve_enabled`): `off` = never auto-activate, `substantive` = auto-activate at the four cognition points when the session's weight class is SUBSTANTIVE. The `AGENTCTL_ADVISOR` env var overrides in both directions (`1` forces on, `0` forces off); a missing key resolves to off. Cost bound per call: `--model sonnet` + hard 20 s subprocess timeout (`advisor._ADVISOR_TIMEOUT_S`). |
| `principle-promotion-threshold` | `3` | Occurrence count (Σ `### ` context blocks across an experience-leaf cluster sharing one functional ground) at/above which `record-experience.py promote-scan` flags the cluster as a candidate to lift into a principle/v1 leaf. Distinct from `core-difficulty-mass-threshold` (severity-mass for Core edits) — this is a **recurrence count** for principle induction. Basis: the Rule of Three (Fowler, *Refactoring* 2nd ed.) — two occurrences may be coincidence; a third establishes a recurring pattern worth generalizing. Tunable here. |
| `skill-description-max-chars` | `850` | Hard ceiling on a `SKILL.md` frontmatter `description:` value — always-visible index cost (loaded into every session's skill list), unlike the skill body, which loads only on invocation. Measured 2026-07-23: the longest of the 11 current skill descriptions is 793 chars (`skills/specializations/developer/SKILL.md`), so `850` leaves headroom while still catching runaway growth. Enforced by `scripts/lint-prose-length.py`. |
| `always-loaded-surface-advisory-chars` | `80000` | ADVISORY-only (never FAIL) ceiling on the aggregate always-loaded surface — CLAUDE.md + config.md + `memory-global/MEMORY.md` + the sum of every skill's frontmatter description. Report-only via `scripts/lint-prose-length.py --surface-report`; deliberately not enforced, because an aggregate ceiling incentivizes deleting someone else's rule to stay under budget rather than fixing your own contribution (the 2026-07-03 regression this guards against). Measured 2026-07-23 aggregate ≈ 75 700 chars (via `scripts/lint-prose-length.py --surface-report`). |

Audit cross-references with `rg '<key>' --glob '*.md' ~/claude-agent-instructions/` to see who depends on each value.
