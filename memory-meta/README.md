# Agent memory (вне Arcadia)

Локальная «память» агента: факты, пайплайны и выводы, которые **не** должны раздувать общие `CLAUDE.md` и промпты sub-агентов.

**Путь:** `~/.claude/memory/` (тот же каталог для Cursor и Claude Code — см. `~/.claude/CLAUDE.md`).

## Как пользоваться

1. Начни с **[INDEX.md](INDEX.md)** — оглавление и «общая картина» по темам.
2. Читай только релевантные leaf-файлы; не подгружай всю память в контекст.
3. Файлы ссылаются друг на друга относительными путями (`[текст](deepagent/....md)`).

## Когда писать в память

Агент **предлагает** пользователю «запомнить» открытие, если:

- разобрали нетривиальный prod/исторический пайплайн;
- исправили неверное допущение, которое может повториться;
- зафиксировали термин, схему тикетов или контракт между задачами.

Пользователь подтверждает — агент создаёт/обновляет файл в `memory/` и строку в `INDEX.md`.

## Агенты memory / self-improvement

- Запись и навигация по этому каталогу — субагент **memory** (`~/.claude/agents/memory.md`).
- Улучшение промптов, CLAUDE.md, git-репозитория инструкций — субагент **self-improvement**.
- Координация — **manager**; краткие правила делегирования — в `~/.claude/CLAUDE.md`.

## Структура

```
memory/
  README.md          ← вы здесь
  INDEX.md           ← оглавление
  deepagent/         ← темы по репозиторию robot/deepagent
    ...
```

Новые верхнеуровневые темы — новые папки или файлы в `INDEX.md`.
