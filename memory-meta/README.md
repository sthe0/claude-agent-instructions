# Agent memory (вне Arcadia)

Локальная «память» агента: факты, **доменные runbook'и**, пайплайны и выводы, которые **не** должны раздувать общие `CLAUDE.md` и промпты generic sub-агентов (**manager**, **yandex-developer**, **planner**).

**Путь:** `~/.claude/memory/` (тот же каталог для Cursor и Claude Code — см. `~/.claude/CLAUDE.md`).

## Как пользоваться

1. Начни с **[INDEX.md](INDEX.md)** — оглавление и «общая картина» по темам.
2. Читай только релевантные leaf-файлы; не подгружай всю память в контекст.
3. Файлы ссылаются друг на друга относительными путями (`[текст](deepagent/....md)`).
4. При чтении leaf — **ленивая проверка актуальности** (см. ниже).

## Актуальность памяти

Устаревшая память хуже отсутствия памяти: агент действует по неверному runbook. Поддерживай freshness явно.

### При записи (обязательно для runbook и контрактов API/CLI)

В каждом leaf с процедурой или именами в коде добавляй блок **`## Метаданные`** (таблица):

| Поле | Смысл |
|------|--------|
| `last_verified` | Дата (ISO), когда факты сверены с кодом/прогоном |
| `staleness_triggers` | Что сделает запись недостоверной (рефактор модуля, смена CLI, удаление кубика) |
| `revalidate` | Как проверить за 2–5 минут: `grep`/путь в Arcadia, команда, ссылка на файл |

Без `revalidate` leaf не считается готовым к долгому хранению.

### При чтении (лениво)

Перед тем как **опереться** на leaf в решении:

1. Прочитай `## Метаданные` и `revalidate`.
2. Если прошло много времени, есть `staleness_triggers` по текущей задаче, или тикет трогал тот же код — **выполни минимальную revalidate**.
3. При расхождении — обнови leaf и `last_verified`, либо спроси пользователя; устаревшее удаляй или помечай «устарело».

### Периодическая уборка

По запросу пользователя или раз в несколько недель (субагент **memory**): пройти INDEX, revalidate старые runbook-leaf, убрать мёртвое.

## Когда писать в память

Агент **предлагает** пользователю «запомнить» открытие, если:

- разобрали нетривиальный prod/исторический пайплайн или runbook перезапуска (какие стадии не трогать при ретесте);
- пользователь сказал «запомни» или отклонил перенос факта в промпт агента как «слишком конкретно»;
- исправили неверное допущение, которое может повториться;
- зафиксировали термин, схему тикетов или контракт между задачами.

Пользователь подтверждает — агент создаёт/обновляет файл в `memory/` и строку в `INDEX.md` (**с блоком метаданных**, если запись может протухнуть).

## Агенты memory / self-improvement

- Запись и навигация по этому каталогу — субагент **memory** (`~/.claude/agents/memory.md`).
- Улучшение промптов, CLAUDE.md, git-репозитория инструкций — субагент **self-improvement**.
- Координация — **manager**; краткие правила делегирования — в `~/.claude/CLAUDE.md`.

## Структура

```
memory/
  README.md          ← вы здесь
  INDEX.md           ← оглавление
  deepagent/         ← темы по репозиторию robot/deepagent
    ...
```

Новые верхнеуровневые темы — новые папки или файлы в `INDEX.md`.
