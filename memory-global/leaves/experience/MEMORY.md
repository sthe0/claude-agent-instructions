## 2026-06

- [2026-06-25 — hook-state-gate authorizes by the ACTING session's engine node, reached only via a .toml plan](2026-06-25-state-gate-needs-acting-session-at-executing-via-toml.md) — Production Edit/Write is denied unless the engine state of the session that issues the write is at an execution node (EXECUTING). Two non-obvious consequences: (1) markdown plans are structure-verified but do NOT populate state.stages, so next-stage/dispatch never reach EXECUTING — only a .toml plan populates stages; (2) a spawned developer runs under its OWN fresh, unclassified session, so the parent being at EXECUTING does not authorize the child — the child is denied every write and burns its whole budget before dying.
- [2026-06-24 — Spawned developer's return marker after a prose preamble false-BLOCKs the engine](2026-06-24-developer-marker-not-on-line-1-false-block.md) — Recurring engine friction — a spawned specialist (developer) writes a summary paragraph before its COMPLETED:/PLAN-READY: return marker, but the spawn wrapper only accepts the marker on the first non-empty line, so it prefixes MALFORMED: and agentctl routes the stage to BLOCKED even though the work succeeded. The manager must then independently re-verify, unblock, and record-result. Fix lives in the wrapper (tolerate a marker on any line / scan the tail) and/or the developer SKILL (marker strictly on line 1).
- [2026-06-24 — A process-gate exemption is suspect when the exempted artifact is itself a gated result-image](2026-06-24-gate-exemption-is-category-error-for-result-images.md) — Difficulty — a path is blanket-exempted from a process-gate because it 'isn't task output', but the artifact is in fact the result-image of a process the same gate governs (a plan is the output of planning). The unconditional exemption is then a category error: it lets the artifact be mutated in states where mutating it is exactly the thing the gate exists to catch. Fix: gate it with a phase/node-aware rule (writable only in the producing phase, frozen after), and resist weakening the gate to preserve an existing escape hatch — route the escape through the difficulty-overcoming path instead.
- [2026-06-24 — Porting a prose process to code: secure a real consumer, don't collapse a prose superset](2026-06-24-prose-to-code-migration-consumer-and-superset.md) — Difficulty — moving a prose coordination process into agentctl code fails two ways: the new code has no runtime consumer (advisory dead code, e.g. a pure function nothing calls), or it is narrower than the prose it 'replaces' (information loss). Fix: (a) give the function a real consumer today, ideally an existing duplicated implementation it can delegate to; (b) before collapsing prose to a code pointer, check whether the prose is a richer superset — if so cross-reference, don't delete.
- [2026-06-11 — Migrating populated native auto-memory into the project tree](2026-06-11-project-memory-migrate-existing-native.md) — Wiring project-tree memory on a machine that already has populated ~/.claude/projects/<hash>/memory: the stock setup-project-memory.sh stubs the live memory and moves it to .bak, orphaning accumulated content. Migrate content first, then swap the native path to a symlink.
- [2026-06-04 — Verify the load-bearing axis, not the surface signal](2026-06-04-verify-load-bearing-axis.md) — recurring difficulty: a proxy (daemon `active`, a confident research claim, a green static check, a prior plan's assumption) is trusted while the axis that carries the conclusion disagrees. Contexts: token-economy (web claim vs primary docs; measurement overturned the static-prefix plan) + ccgram-ENOSPC (active daemon, dead background loops). Fix: observe the load-bearing fact directly.
- [2026-06-04 — Org-specific vs. global instruction placement](2026-06-04-org-specific-vs-global-placement.md) — org-specific dispatch rules drafted into global (org-agnostic) instructions; the rule existed but didn't fire. Plus the arc-tracked `junk/` symlink topology making deepagent project memory read "outside repository" from the wrong mount. Test: would it make sense off-Yandex?
- [2026-06-04 — Non-English skill content vs. verify-language; the CLAUDE.md cap](2026-06-04-non-english-skill-content-language-wall.md) — a skill whose working material is non-English hits verify-language's per-line within-3-lines wall (one note can't cover a Cyrillic table → fenced/backtick spans are stripped). Plus the recurring CLAUDE.md 400-line cap collision when adding specialization rows.

## 2026-05

- [2026-05-29 — Logical-pwd self-locating script footgun (ELOOP)](2026-05-29-logical-pwd-self-locating-script.md) — a script that derives its base dir via logical `pwd`, invoked through a symlink it then rewrites, self-linked `.claude` (ELOOP) and broke a ticket worktree. General fix: enumerate reality and assert it against the description.

Fix: a mechanically-nudged gate shaped to the criterion type (acceptance-review needs a *specific observed* outcome).
